Add a Sauter quadrature task to the ring buffer, which is called by a producer thread.

Note
About the scratch_data argument
  1. It data fields for permuted mapping support points in \(K_x\) and \(K_y\) should have been computed for a pair of DoFs on the CPU.
  2. It is created with the creation of the associated producer thread. During its initialization, a CUDA stream is created for this producer thread.
  3. When adding a Sauter quadrature task, its data fields for permuted mapping support points in \(K_x\) and \(K_y\) will be copied to the related data fields in the ring buffer.
  4. For other quadrature related values, namely Jacobians, normal vectors, covariant matrices and quadrature point coordinates in real cells, they will be computed on the device when the quadrature task is fetched from the ring buffer. The results will be stored back into the ring buffer.
Parameters
fullmatFull matrix within a near field \(\mathcal{H}\)-matrix node
fullmat_row_indexRow index of the entry in the full matrix
fullmat_col_indexColumn index of the entry in the full matrix
i_local_index
j_local_index
mass_matrix_entryValue of the mass matrix entry, if there is any.
scratch_data

Create the new task:

  1. Copy the pointer to the current full matrix, matrix row and column indices and mass matrix entry.
  2. Copy permuted mapping support points in \(K_x\) into ring buffer on the host.
  3. Copy permuted mapping support points in \(K_y\) into ring buffer on the host.
  4. Copy other scalar data.

template<int capacity, int dim, int spacedim, typename RangeNumberType , typename KernelNumberType >
bool CUDAWrappers::SauterQuadratureTaskRingBuffer< capacity, dim, spacedim, RangeNumberType, KernelNumberType >::if_all_near_field_hmat_entries_added

Flag indicating if all matrix entries in all near filed \(\mathcal{H}\)-matrices have been added as tasks to the ring buffer. If it is true, even though the number of tasks in the buffer (task_num) is less than the total number of GPU threads specified to compute the Sauter quadrature, the consumer threads will get all of them directly and perform the computation.

Note
This flag will be set to true when all producer threads have exited. After that, the consumer threads will be notified to fetch and process the remaining tasks in the buffer, the number of which may be smaller than the batch size.
