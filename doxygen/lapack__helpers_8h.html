Detailed Description

Exposes LAPACK helper functions defined in lapack_full_matrix.cc and define new ones by following them as examples.

Note
Because helper functions in lapack_full_matrix.cc are template functions and they are not defined in a header file, their instantiation is limited within the translation unit lapack_full_matrix.cc. Therefore, the helper functions are not available in the deal.ii dynamic library compiled in Release mode. But why they are available in the Debug mode library is still a mystery. Real valued version of the helper function for trsv, which solves a unit or non-unit, upper or lower triangular matrix.

Note
1. Access the matrix data: the internal data contained within a LAPACKFullMatrixExt has the type AlignedVector, which is a protected member variable in the parent class TableBase of LAPACKFullMatrixExt. Therefore, this variable can be directly visited from LAPACKFullMatrixExt and passed into this function.
  1. Access the RHS vector data: being different from LAPACKFullMatrixExt, the values member variable of type AlignedVector contained in the RHS vector is private, which cannot be accessed from LAPACKFullMatrixExt. Therefore, the pointer to the internal data within the RHS vector can be obtained by calling the member function Vector::data(). Then it is passed to this function.
