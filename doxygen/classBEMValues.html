Detailed Description

template<int dim, int spacedim, typename RangeNumberType = double>
class BEMValues< dim, spacedim, RangeNumberType >

Values related to a pair of cells (panels) used in Galerkin BEM, which can be considered as a counterpart of the FEValues for FEM in deal.ii.

Note
The values encapsulated in BEMValues are shape functions value and their gradient values at each quadrature point, as well as Sauter quadrature rules for different cell neighboring types. These values will be precalculated for improving the performance.

template<int dim, int spacedim, typename RangeNumberType >
void BEMValues< dim, spacedim, RangeNumberType >::resize_internal_data_in_mappings ( const unsigned int  n_q_points) const
protected

Resize the internal data in all orders of mappings.

Note
The InternalData within a MappingQ object is a temporary place holding the shape function values and their derivatives. Because deal.ii is an FEM library, which does not handle different \(k_3\) terms and cell neighboring types as in the Sauter quadrature used in BEM, whenever we come to a kind of cell neighboring type which has its associated quadrature object, we need to reinitialize the InternalData objects in all possible orders of mappings and then compute shape function values and their derivatives. After the computation, the values within InternalData will be copied into related data tables in BEMValues.

◆ shape_function_values_common_edge()

template<int dim, int spacedim, typename RangeNumberType >
void BEMValues< dim, spacedim, RangeNumberType >::shape_function_values_common_edge

Calculate the table storing shape function values and derivatives for both finite element and mapping objects at Sauter quadrature points for the common edge case.

Initialize the internal data in the mapping objects.

Quadrature points in the Sauter's parametric space.

Get the polynomial space inverse numbering for accessing the shape functions in the lexicographic order.

Here I have adopted an assumption that the finite elements are based on tensor product polynomials.

Quadrature points in the unit cells of \(K_x\) and \(K_y\) respectively.

Iterate over each finite element shape function in the lexicographic order on the unit cell of \(K_x\) and evaluate it at the current quadrature point kx_unit_quad_points[q].

Iterate over each finite element shape function in the lexicographic order on the unit cell of \(K_y\) and evaluate it at the current quadrature point ky_unit_quad_points[q].

Compute mapping shape function values and their derivatives in batch.

Even though the internally generated polynomials in the mapping object are in the tensor product order, the shape function values and derivatives are still in the hierarchic order. This can be verified by checking the source code of MappingQ<dim, spacedim>::InternalData::compute_shape_function_values. (see http://localhost/dealii-9.4.1-doc/mapping__q_8cc_source.html#l00271)

However, this behavior is different from the documentation for the function MappingQ< dim, spacedim >::InternalData::shape().
